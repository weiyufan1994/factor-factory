# 因子迭代方法论知识库

> 本文档记录因子增强与市场理解的通用规律，供新因子研究时举一反三。
>随研究进展持续更新。

---

## 一、因子增强核心手法

### 1. Fold（折叠）变换

**适用场景**：原始 signal 方向混乱，G10 为负或接近零，但 IC 方向正确。

**操作方法**：
```
原始 signal = s
folded = -abs(s)
```
等价于：`((condition) ? -ts_rank(abs(...)) : 0)`

**原理**：
- `abs()` 把方向抹掉，只留幅度
- 负号把"异动大→做空"变成"异动大→做多"
- 核心洞察：`ts_rank(delta)` ≠ `ts_rank(|delta|) × sign`，不能代数等价替换，必须实测

**案例**：Alpha007 Step1→Step2，IC 0.047→0.068，G10 从负转正。

---

### 2. Regime 加权（自适应权重）

**适用场景**：因子在某些市场状态下有效（crash regime），在其他状态下失效（bubble regime），表现为 G10 尾部不稳定、倒V陷阱。

**操作方法**：
```
regime = 1 + alpha * (kurtosis_zs - skewness_zs)
final_signal = base_signal * regime
```

**Regime 变量计算**：
| 变量 | 公式 | 含义 |
|------|------|------|
| skewness | `rolling(20d_return, 60d).skew()` | 负偏=crash，正偏=bubble |
| kurtosis | `rolling(20d_return, 60d).kurt()` | 肥尾程度 |
| kurt - skew | `kurtosis_zs - skewness_zs` | crash强度（越高越极端） |

**机制**：
- kurtosis 高 → 两边肥尾都存在 → regime 高 → 信号放大
- skewness 负（crash）→ kurt - skew 更大 → 放大更显著
- skewness 正（bubble）→ kurt - skew 抵消 → 信号温和

**注意**：
- `kurt - skew` 优于 `kurt + |skew|`（有方向逻辑，rho 0.988 vs 0.952）
- `|skew|` 优于 raw skew（只关心幅度，不区分方向；raw skew 在 bubble 时过度压制信号）
- 参数 alpha=0.5 由经验确定，未经严格分样本验证（新因子研究时可作为调参方向）

**案例**：Alpha007 Step4→Step6，rho 0.758→0.988，G10 +0.014%→+0.022%

---

### 3. TsRank + rank 组合

**发现**：单独用 `rank(mcap)` IC=0.068；用 `TsRank(rank(mcap))` IC=0.072。

**原理**：
- `rank` 是静态截面排序
- `TsRank` 是滚动时序排序，引入了趋势持续性信息
- 两者叠加比单独使用更稳健

---

### 4. 双重维度叠加（combo）

**适用场景**：单维度因子效果已达瓶颈，需要寻找新的收益来源。

**操作方法**：
```
combo = (负向信号A + 负向信号B) / 2
做多：同时满足 A好+B好
做空：同时满足 A差+B差
```

**关键洞察**：
- "双重优良"或"双重恶劣"叠加，效果大于各自独立
- combo 的优势往往来自空头侧（G01），而非多头侧
- 中性化测试（neutralize市值）几乎不影响结果 → 说明两个维度效应独立

**案例**：
- `(-TsRank(rank(mcap)) + -TsRank(rank(vol))) / 2` → IC=0.072，spread +0.380%/天
- 空头侧 G01=-0.309%，强于任何单因子空头

---

### 5. 市值中性化测试

**操作**：`factor_neutralized = neutralize(factor, by=[mcap])`

**判断标准**：
- 中性化后 IC 几乎不变 → 该因子的收益来源独立于市值
- 中性化后 IC 明显下降 → 因子收益主要来自市值暴露

---

### 6. Decile 单调性修复（进阶）

**适用场景**：G10/G01 单调，但 G02-G09 中间区间有凹凸（倒V）。

**Regime 维度**：
- 观察 crash regime（偏度为负）和 bubble regime（偏度为正）下因子表现差异
- 如果只在 crash 时失效 → 引入 regime 权重（如上方法）

## 二、Read the Market 框架

### 1. Decile 分组分析

每个因子测试必做：
```
G01 = top 10% 账面市值比（最value）
G10 = bottom 10%（最growth）
```
观察：
- 完美单调：从负到正，无倒V
- 倒V陷阱：G10 反而变负，说明因子在极端growth股票上失效

**单调性指标**：Spearman rho + p-value
- rho > 0.9 + p < 0.001 → 完美单调
- rho > 0.8 + p < 0.01 → 良好单调
- rho < 0.7 → 单调性不足，需要修复

---

### 2. 多空收益分解

```
spread = G10 - G01（日均）
```
- spread > 0：多头有效、空头也有效
- G10 > 0 但 G01 ≈ 0：纯多头，空头是噪音
- G01 < 0 但 G10 ≈ 0：纯空头，多头是噪音

**判断因子质量**：
- 真正好的因子：G10 和 G01 方向明确，spread 宽
- 有问题的因子：IC 高但 G10 ≈ 0（靠空头撑起 IC）

---

### 3. Regime 识别

| 市场状态 | 偏度（skewness） | 峰度（kurtosis） | 典型事件 |
|---------|----------------|-----------------|---------|
| 正常 | 接近0 | 接近3 | — |
| Crash | 负偏 | 高 | 上海疫情 2022-03，贸易战 2018 |
| Bubble | 正偏 | 高 | 2015杠杆牛 |

---

### 4. 因子失效应识别

**立即标记为 FAIL 的信号**：
- IC ≈ 0（无信号）
- G10 ≈ 0 且 G01 ≈ 0（spread 来自幸存者偏误）
- 单调性 rho < 0.5

**需要 iterate 的信号**：
- G10 为正但不稳定 → 检查 crash regime
- IC 不错但 p-value 不显著 → 样本量不足或信号过弱
- NAV 回撤期集中 → regime 加权或缩短持有周期

## 三、因子决策树（新因子研究时参照）

```
收到新因子
  │
  ├─ 运行基础 IC/IR 测试
  │     ├─ IC ≈ 0 → FAIL，reject
  │     └─ IC > 0 → 继续
  │
  ├─ 运行 Decile + monotonicity 测试
  │     ├─ 完美单调（rho>0.9）→ 升正式库
  │     └─ 有倒V或单调性不足 → 尝试 Fold 变换
  │
  ├─ Fold 变换后
  │     ├─ IC 提升 + 单调性改善 → 尝试 Regime 加权
  │     └─ Fold 后仍无效 → 尝试 combo 或 reject
  │
  ├─ Regime 加权（kurt-skew）
  │     ├─ rho 提升至 0.9+ → 升正式库
  │     └─ 提升不足 → 结合 combo 测试
  │
  └─ Combo 测试
        ├─ IC + spread 提升 → 升正式库
        └─ 无提升 → reject 或归档为低优先级
```

## 四、迭代治理规则：防止把调参误认为研究

已有的 Fold、Regime 加权、TsRank+rank、combo、中性化等方法，是经验工具，不是自动胜利按钮。

每次使用这些方法前，Step6/Researcher 必须先回答三件事：

1. **收益来源**：这次修改强化的是风险溢价、信息优势、约束驱动套利，还是只是改漂亮了 metric？
2. **泛化理由**：为什么这个修改应当跨窗口、跨 regime、跨 universe 更稳，而不是只适配上一轮样本？
3. **反证标准**：下一轮如果哪些指标不改善，就停止这个修改方向？

推荐记录字段：

```text
revision_operator: fold|regime_weight|tsrank_rank|combo|neutralize|winsorize|smooth|portfolio_cost_control|other
target_return_source:
generalization_argument:
overfit_risk:
kill_criteria:
idea_seed:
transferable_pattern:
anti_pattern:
```

### 不允许的研究行为

- 只因为 IC 或 spread 提升，就直接把修改认定为成功。
- 连续尝试多个 wrapper，只保留最好结果，不记录失败路径。
- 把 Fold、Regime、Combo 当成万能修复器。
- 明明是局部 feature experiment，却写成稳定因子。
- 没有解释对手盘、约束来源或风险补偿，就进入正式库。

### 鼓励的研究行为

- 从失败中提取反向因子、状态因子或组合因子的 idea seed。
- 把有效 modification 抽象为可迁移 pattern，而不是只贴在某个 report_id 下。
- 对相似因子先检索历史知识库，再决定是否复用、反用或绕开旧方法。
- 记录“这条路不要再试”的 anti-pattern，节约未来研究预算。

## 五、研究员学习与创新机制

因子工厂的目标不是积累一堆回测文件，而是培养一个会学习的 researcher agent。

每个 case 结束后，知识库至少应沉淀四类内容：

| 类型 | 要回答的问题 | 示例 |
|------|--------------|------|
| Pattern | 什么情况下这种因子有效？ | 放量异动在 crash-like regime 下更适合 fold/reversal |
| Anti-pattern | 什么做法看似合理但失败？ | 单纯提升 IC 但 G10 不赚钱，说明多头不可交易 |
| Transfer | 经验能迁移到哪里？ | Alpha007 的 fold 思路可迁移到其他 signed abnormal-flow 因子 |
| Idea seed | 下一代创新方向是什么？ | 用 regime state 动态选择 fold / raw / neutralized signal |

未来的我、Bernard 和 Humphrey 在研究新因子时，应优先做：

1. 检索相似因子与失败案例；
2. 识别可迁移 pattern 与 anti-pattern；
3. 提出至少一个创新 idea seed；
4. 再决定是否重跑 Step3B 或新开一个 factor experiment。

一句话原则：

> 每次研究都要让下一次研究更聪明。

## 六、通用失败模式清单

| 失败模式 | 典型表现 | 处理方式 |
|---------|---------|---------|
| 信号方向混乱 | G10 负或接近0 | Fold 变换 |
| Crash regime 失效 | G10 不稳定，NAV 回撤 | Regime 加权 |
| 纯空头因子 | G10≈0，G01负 | 评估实现成本，决定是否保留 |
| IC 虚高 | G10 接近0，靠空头撑IC | 拆解多空收益，确认多头真实存在 |
| 公式代数等价替换错误 | Fold 后效果不符预期 | ts_rank(signed) ≠ ts_rank(abs) × sign，需实测 |
| 市值暴露主导 | 中性化后 IC 下降明显 | 保留市值暴露结论，作为 size 因子处理 |
